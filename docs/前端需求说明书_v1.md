# AI 漫剧制作平台 — 前端需求说明书 v1

> 版本：v1（2026-06-20）
> 对应：前端展示方案_v2 + 自研方案_v5 章节 5.5~5.6 / 6.2~6.4 / 6.6

---

## 一、项目背景

AI 漫剧制作平台是一个面向创作者的剧本→视频全管线自动化工具。用户输入一个故事，系统自动完成剧本创作、分镜设计、角色定妆、逐帧出图、转视频、加字幕、合成最终片的全流程。

前端作为**用户唯一操作界面**，需要：
- 实时展示管线各环节的进度和中间产出
- 在关键审核节点提供人工干预能力
- 支持历史任务管理和资产复用

---

## 二、技术约束

| 项 | 要求 |
|:---|:---|
| 架构 | 前后端分离，前端纯静态，不依赖后端模板引擎 |
| 技术栈 | 纯 HTML + CSS + Vanilla JS（零构建工具，改完刷新即生效） |
| 通信 | REST API（任务提交/查询）+ SSE（实时推送事件） |
| 文件服务 | 后端通过 FastAPI StaticFiles 托管图片/视频/SRT |
| 部署 | 前端文件放在 FastAPI StaticFiles 目录下，与后端同端口 |
| 浏览器 | 现代浏览器（Chrome/Firefox/Edge 最新版） |

## 五、功能详述

### 5.1 输入区

**位置**：主内容区最上方，7 卡片之前

**包含元素**：
- **故事输入框**：textarea，多行输入，placeholder="键入你的故事，如：一个程序员穿越到全是猫娘的世界..."
- **风格选择器**：select/下拉框，选项：写实 / 二次元 / 水墨风，默认"写实"
- **开始制作按钮**：紫色渐变背景，点击后变灰禁用，更新文字为"⏳ 执行中…"
- **停止按钮**：红色背景，默认隐藏，执行中显示

**交互逻辑**：
- 点击"开始制作"时，向后端发送 `POST /api/pipeline/run`
- 请求体：`{ "story": "...", "style": "realistic|anime|ink_wash" }`
- 成功后锁定输入框和按钮，显示停止按钮
- 停止按钮调用 `POST /api/pipeline/cancel/{id}`

### 5.2 7 卡片全链路展示

**布局**：一行横排 7 张卡片，每张代表一个管线 Agent

**卡片结构**：
```
┌──────────────────┐
│     ✅          │  ← 状态徽标（按需显示）
│     📝 剧本       │  ← 图标 + 标题
│     3集已生成     │  ← 状态文字
└──────────────────┘
```

**7 个 Agent 对应**：

| 索引 | 标题 | 空闲文字 | 内部进度 |
|:---|:---|:---|:---|
| 1 | 📝 剧本 | "空闲" | "第X集/共Y集" |
| 2 | 📋 分镜 | "空闲" | "第X集 X/Y镜" |
| 3 | 🎭 定妆照 | "空闲" | "已生成X个角色" |
| 4 | 🖼️ 出图 | "空闲" | "X/Y张" |
| 5 | 🎬 视频 | "空闲" | "X/Y段" |
| 6 | 📄 字幕 | "空闲" | "X/Y集" |
| 7 | 📦 合成 | "空闲" | — |

**状态文字更新规则**：
- SSE 收到 `status` 事件时 → 更新对应卡片的 `.status-text`
- `"agent": "image", "step": 5, "total": 20` → 文字变"5/20张"

### 5.3 全局进度条

**位置**：7 卡片下方

**组成**：
- 渐变色填充条（紫色→青色）
- 右侧百分比文字
- 下方状态提示行

**规则**：
- 百分比 = (已完成 Agent 数 / 7) × 100%
- SSE `status` 事件更新百分比
- **当有卡片处于 state-blocked 时**：进度条动画暂停，文字切换为红色：
  `"⏸️ 等待人工确认：请检查 [定妆照/分镜]"`
- 内部进度（如出图 12/20 张）在卡片上单独显示

### 5.5 左侧面板

#### 5.5.1 Tab 切换

顶部两个 tab 按钮：`🏛️ 资产库` / `📋 任务概览`，点击切换内容区域。

#### 5.5.2 资产库 Tab

**数据来源**：`GET /api/assets/list?pipeline_id={id}`

**内容**：
- **角色定妆照**：网格展示，每个角色占一个卡片
  - 头像缩略图（64px 见方）
  - 角色名
  - 复用次数徽章（如 x3）
  - hover 放大
- **场景背景**：同上，显示场景名称和引用次数

#### 5.5.3 任务概览 Tab

**内容**：
- 剧名
- 总集数
- 总分镜数
- 当前状态（空闲/执行中/已完成）
- 已耗时 + 预估剩余时间
- Checkpoint 列表：每个 Agent 一个状态点（✅/⏳/▫️）

**关闭**：右上角 ✕ / 点击遮罩

- 剧名

**位置**：详情区下方
**可展开/收起**：默认展开，最大高度 100px

**格式**：
```
16:45:25 [Agent2] ✅ 分镜校验通过，共22个shot
16:45:16 [Agent2] DeepSeek调用中..., prompt_len=1573
```

**颜色规则**：
- 灰色：普通 INFO
- 绿色：SUCCESS 级别  
- 红色：ERROR 级别
- 黄色：WARN 级别

**交互**：最新日志显示在最上面，超过 30 条移除最旧条目

## 六、SSE 事件协议

### 6.1 事件列表

| 事件名 | 触发时机 | 前端处理 |
|:---|:---|:---|
| `log` | 每步操作 | 日志区追加一行，附颜色 |
| `status` | Agent 内部进度更新 | 更新卡片计数 + 全局进度条 |
| `agent_done` | 单个 Agent 完成 | 卡片切换状态 + 进度条更新 |
| `agent_blocked` | Agent 需人工确认 | 卡片切换为 state-blocked，进度条暂停，显示审核横幅 |
| `shot_done` | 单张图/单段视频完成 | 实时生成流插入缩略图（左侧滑入） |
| `pipeline_done` | 全管线完成 | 所有卡片更新，切换到合成视图 |

### 6.2 SSE 数据格式（JSON）

```
event: log
data: {"time":"16:45:12","agent":"script","level":"INFO","message":"DeepSeek调用中..."}

event: status
data: {"agent":"image","step":5,"total":20}

event: agent_blocked  
data: {"agent":"storyboard","reason":"awaiting_review","shots":20}

event: shot_done
data: {"agent":"image","shot_index":5,"total":20,"url":"/storage/ep1_shot5.png"}

event: agent_done
data: {"agent":"image","success":true,"summary":{"total":20,"shot_ids":[...]}}

event: pipeline_done
data: {"id":"20260620_1615","total_time":"00:18:32"}
```

## 附录 A：术语表

| 术语 | 说明 |
|:---|:---|
| Agent | 管线中的单个处理节点（剧本/分镜/定妆照/出图/视频/字幕/合成） |
| Pipeline | 从输入到最终产出的完整任务流程 |
| Shot | 一个分镜镜头，也是出图和视频的最小单位 |
| Episode | 一集内容，含多个 scene / shot |
| SSE | Server-Sent Events，服务端向浏览器推送事件 |
| State | 卡片状态（idle/pending/running/blocked/done/fail） |
| Blocked | 待人工审核状态 |
| 定妆照 | 角色形象的确定设计 |
| 实时生成流 | 出图/视频生成中实时展示每个完成的缩略图 |

## 附录 B：前端基准则

1. **纯静态，零构建**：HTML + CSS + JS，不引入 npm/node_modules 任何依赖
2. **前后端分离**：不适用模板引擎，所有数据通过 REST/SSE 获取
3. **不复制现有代码**：所有前端代码独立编写
4. **单页应用**：所有视图在同一个页面中通过 JS 切换
5. **API 基址可配**：通过 `app.js` 顶部的 `API_BASE` 常量控制
6. **开发模式**：无后端时可通过 mock 数据独立运行和调试
