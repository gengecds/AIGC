# AI 漫剧制作管线 — 前端展示方案

> 对应自研方案_v5 章节：5.5~5.6 / 6.2~6.4 / 6.6
> 版本：v2（2026-06-20）

---

## 一、目标

**让用户看到管线全流程的实时状态、中间产出（文案/图片/视频）、并能干预关键节点（审核文案、调整分镜）。**

---

## 二、技术选型

### 架构原则：前后端分离

前端纯静态页面（HTML + CSS + JS），不依赖后端模板渲染。
前端通过 REST API 与后端通信，通过 SSE 接收实时事件。
后端只提供数据接口和静态文件服务，完全独立于前端。

| 层 | 技术 | 理由 |
|:---|:---|:---|
| **前端** | HTML + CSS + Vanilla JS（单页） | 零构建工具，改完刷新即生效 |
| **通信** | REST API（提交/查询）+ SSE（实时推送） | 前端调后端，不走页面渲染 |
| **后端 REST** | FastAPI（已有） | 任务提交 / 状态查询 / 产出下载 |
| **后端 SSE** | FastAPI SSE (sse-starlette) | 管线进度实时推送 |
| **文件服务** | FastAPI StaticFiles | 图片 / 视频 / SRT 通过 HTTP 静态路径访问 |

### 部署关系

```
用户浏览器
    ↕ REST / SSE
FastAPI 服务 (8888端口)
    ├── /api/*          → 业务接口
    ├── /events/*        → SSE 事件流
    ├── /storage/output  → 静态文件（图片/视频/字幕）
    └── /frontend/       → 静态文件（前端页面）
```

前端页面通过 `FastAPI StaticFiles` 托管，后端不加模板引擎；前端在浏览器中通过 `fetch` + `EventSource` 与后端通信。

## 三、视觉设计规范

### 3.1 整体风格

- **暗色主题**：`#0a0a0f` 极深蓝黑背景
- **卡片**：`#16161f` 底色 + `#2a2a3a` 边框 + 12px 圆角
- **主题色**：`#6c5ce7` → `#00cec9` 紫青渐变
- **字体**：系统栈 `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto`
- **间距**：24px 网格，卡片内 padding 16~20px

### 3.2 颜色系统

| 用途 | 色值 | 说明 |
|:---|:---|:---|
| 页面背景 | `#0a0a0f` | 极深蓝黑 |
| 卡片背景 | `#16161f` | 深色卡片 |
| 卡片边框 | `#2a2a3a` | 弱分割线 |
| 主题色 | `#6c5ce7` → `#a29bfe` | 主色 / hover |
| 主题辅色 | `#00cec9` | 进度条 / 成功高亮 |
| 成功 | `#00b894` | Agent 完成 |
| 失败 | `#e17055` | 报错 |
| 警告 | `#fdcb6e` | 橙色提示 |
| 待审核 | `#ff6b81` → `#ff4757` | 红色呼吸光晕 |
| 文字主 | `#e8e8f0` | 标题 / 正文 |
| 文字次 | `#8888a0` | 辅助说明 |
| 文字淡 | `#555570` | 占位 / 提示 |
| 强调色 | `#fd79a8` | 编辑按钮 |

### 3.3 7 卡片独立配色

| 卡片 | 图标底色 | 强调色 |
|:---|:---|:---|
| 📝 剧本 | `#6c5ce7` 紫 | `#a29bfe` |
| 📋 分镜 | `#0984e3` 蓝 | `#74b9ff` |
| 🎭 角色 | `#e84393` 粉 | `#fd79a8` |
| 🖼️ 出图 | `#00b894` 翠绿 | `#55efc4` |
| 🎬 视频 | `#fdcb6e` 金 | `#ffeaa7` |
| 📄 字幕 | `#636e72` 灰 | `#b2bec3` |
| 📦 合成 | `#d63031` 红 | `#ff7675` |

### 3.4 关键交互细节

| 元素 | 行为 |
|:---|:---|
| 卡片 hover | 上移 4px + 阴影加深 + 边框发光 |
| 执行中 | 边框呼吸脉冲动画（2s 循环） |
| 待审核 | 红色光晕呼吸 + 徽标 ⏸️ |
| 进度条 | 渐变填充 + 百分比文字 + 波浪动效 |
| 日志新行 | 从底部滑入，自动滚动到底 |
| 模态框 | 背景模糊 10px + 缩放 0.95→1.0 |
| 成功标记 | 缩放弹出对勾 |

### 3.5 响应式

- 宽屏（>1400px）：7 卡片横排
- 中屏（1024~1400px）：两行 4+3
- 窄屏（<1024px）：左侧导航纵向排列
- 日志区：固定底部，可展开/收起

## 五、功能详述

### 5.1 输入区（对应 v5 6.4 风格选择）

```
┌──────────────────────────────────────────────────────┐
│ 📖 故事输入                                            │
│ ┌──────────────────────────────────┐ ┌──────────┐    │
│ │ 键入你的故事，如：一个程序员穿越   │ │ 🎨 写实 ▾ │    │
│ │ 到全是猫娘的世界…                 │ │          │    │
│ │                                  │ │ 写实     │    │
│ │                                  │ │ 二次元   │    │
│ │                                  │ │ 水墨风   │    │
│ └──────────────────────────────────┘ └──────────┘    │
│                               [🚀 开始制作]          │
└──────────────────────────────────────────────────────┘
```

- **文本框**：多行输入，placeholder 指引
- **风格下拉框**：写实 / 二次元 / 水墨风，默认选中"写实"
- 提交时 body：`{ "story": "...", "style": "realistic" }`

### 5.2 全链路展示区 — 7 卡片（对应 v5 6.3 待审核状态）

**状态机**：

```
         ┌──────────┐
         │   空闲    │
         └────┬─────┘
              │ 提交任务
         ┌────▼─────┐
         │  排队中   │
         └────┬─────┘
              │ 轮到
         ┌────▼─────┐
         │  执行⏳  │ ← 边框脉冲动画，显示内部进度
         └────┬─────┘
        ┌─────┴──────┐
        ▼             ▼
   ┌─────────┐  ┌─────────┐
   │ ⏸️ 待审核 │  │ ❌ 失败  │ ← 红色 + 重试按钮
   └────┬────┘  └─────────┘
        │ 确认
   ┌────▼────┐
   │ ✅ 已完成 │
   └─────────┘
```

**状态对应的卡片外观**：

| 状态 | 卡片边框 | 徽标 | 文字 |
|:---|:---|:---|:---|
| 空闲 | 灰色 `#2a2a3a` | — | "空闲" |
| 排队中 | 灰色虚线 | ⏳ | "排队中" |
| 执行中 | 对应色 + 脉冲 | ⏳ | 内部进度如 "5/20张" |
| **⏸️ 待审核** | **红色 `#ff6b81` 呼吸** | **⏸️ 待确认** | **"等待审核…"** |
| ✅ 已完成 | 绿色 `#00b894` | ✅ | 统计数据 |
| ❌ 失败 | 红色 `#e17055` | ❌ | 失败原因 |

**卡片内部展示**（以定妆照卡片为例）：

```
┌────────────────┐
│ 🎭 定妆照       │ ← 图标 + 标题
│                │
│ ⏸️ 待确认       │ ← 状态徽标
│                │
│ 等待人工确认…   │ ← 状态文字（红色呼吸）
└────────────────┘
```

已完成后：
```
┌────────────────┐
│ 🎭 定妆照       │
│ ✅             │
│ 林风 雪儿 露娜… │ ← 角色名缩略
└────────────────┘
```

### 5.3 任务进度条

```
←--- 全局进度条 ---→
████████████░░░░░░░░░░ 35%

→ 文字行：执行中 [Agent 2 分镜] 第1集生成中… (5/22镜头)
```

- 百分比 = 已完成 Agent 数 / 总数 × 100%
- **待审核时进度条暂停**，文字变为：**"⏸️ 等待人工确认：请检查分镜"**
- 内部进度单独显示（如出图 12/20 张）

## 六、SSE 事件协议

### 6.1 事件类型

| 事件名 | 触发时机 | 前端行为 |
|:---|:---|:---|
| `log` | 每步操作 | 日志区追加一行 |
| `status` | Agent 内部进度更新 | 卡片更新计数 + 进度条 |
| `agent_done` | 单个 Agent 完成 | 卡片状态切为 ✅ 或 ⏸️ 待审核 |
| `agent_blocked` | Agent 需要人工确认 | 卡片切为 ⏸️ 待审核，进度条暂停，显示审核操作栏 |
| `shot_done` | 单张图片/单段视频完成 | 实时生成流插入缩略图 |
| `pipeline_done` | 全管线完成 | 卡片全部更新，显示合成结果 |

### 6.2 SSE 数据格式

```
event: log
data: {"time": "16:45:12", "agent": "script_agent", "level": "INFO", "message": "DeepSeek调用中..."}

event: status
data: {"agent": "image_agent", "step": 5, "total": 20, "progress_pct": 25}

event: agent_blocked
data: {"agent": "storyboard_agent", "reason": "awaiting_review", "episode": 1, "shots_total": 20}

event: shot_done
data: {"agent": "image_agent", "shot_index": 5, "total": 20, "image_url": "/storage/output/ep1_shot5.png"}

event: agent_done
data: {"agent": "image_agent", "success": true, "summary": {"total_images": 20, "shots": [...]}}

event: pipeline_done
data: {"success": true, "pipeline_id": "20260620_1615", "total_time": "00:18:32"}
```

|:---|:---|:---|:---:|
| `/api/pipeline/events/{id}` | GET | SSE 事件流 | — |
